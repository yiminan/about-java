"더 작고 더 단순한 객체를 만들어라"

객체는 단순해질 수록 응집도가 높아지고, 유지보수가 쉬워집니다.
프로그래밍에서 유지보수성은 가장 중요한 요소입니다.
Java에서 허용 가능한 클래스의 최대 크기는 주석 공백 등등 모두를 포함해서 java 파일이 250줄 이하가 되어야합니다.
이미 Open source에서도 5천줄도 넘는 클래스들이 아주 많습니다. 모두 잘못된 클래스입니다.
내가 만든 모든 클래스의 길이가 250줄 이하로 유지된다면 좋은 개발자이고 아키텍트입니다.
여기서 코드란 프로덕션 코드와 테스트 코드 모두를 포함합니다.

일반적으로 불변 객체는 가변 객체보다 클 수 없습니다.
애초에 불변 객체는 생성자로 내부 프로퍼티가 초기화되어야하기 때문입니다.
불변성은 클래스를 더 깔끔하고 짧게 만들 수 있습니다. 이것이 불변 클래스를 구현할 때 얻을 수 있는 가장 중요한 장점입니다.
진정한 객체지향 소프트웨어에는 오직 불변 객체만 존재해야한다고 생각합니다.
가변성은 절차지향 프로그래밍의 끔찍한 유산입니다. 어떤 클래스도 가변 클래스로 만들지 않는게 좋습니다.
